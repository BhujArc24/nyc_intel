The Data

Five datasets, one city.

Every table joins through ZIP code, so you can ask questions that stitch crime, transit, grocery access, and demographics together in a single breath.

01 · Crime
NYPD Complaints
579K+ records with type, severity, coordinates, and demographics across every borough.
02 · Vulnerability
CDC Social Index
Poverty, housing, and language data by ZIP code.
03 · Population
Census ZIPs
Modified ZIP geometry with population.
04 · Groceries
Food Access
Store locations citywide for identifying food deserts and surplus.
05 · Transit
MTA Bus Stops
3,381 stops with shelter and flood data.
✦ The Join
Everything connects by ZIP code
Spatial reverse-geocoding maps lat/lng to ZIPs so questions can cross datasets freely.
How it works

From question to insight in four steps.

No SQL, no dashboards to build — the AI handles translation and the frontend handles the pretty pictures.

1

You ask

Type a question in plain English. Anything from "top crimes in Queens" to "food deserts near high-crime ZIPs."

2

AI translates

GPT-4o-mini turns it into a precise SQL query using the schema, join rules, and borough aliases.

3

Databricks runs it

The query hits real NYC data with auto-retry if something edge-cases out.

4

Visuals appear

Results render as KPI cards, maps, charts, and a plain-English summary — instantly.
